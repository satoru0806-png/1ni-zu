# 夢ナビ Voice AI 設計図

## コンセプト

Typelessの音声AI整形機能を夢ナビに統合。
「話すだけで、きれいな文章になる」体験を実現する。

---

## Typeless機能の再現マッピング

| Typeless機能 | 夢ナビ実装 | 技術 |
|-------------|-----------|------|
| 音声認識 | Web Speech API | ブラウザ内蔵（無料・オフライン可） |
| フィラー除去 | AI整形API | Claude API Haiku |
| 言い間違い修正 | AI整形API | Claude API Haiku |
| 自動句読点 | AI整形API | Claude API Haiku |
| 自動改行 | AI整形API | Claude API Haiku |
| トーン適応 | コンテキスト別プロンプト | 入力先に応じて切替 |
| 学習機能 | ユーザー辞書DB | よく使う単語を記憶 |

## API設計

### POST /api/voice

**リクエスト:**
```json
{
  "rawText": "えーと今日は、あの、企画書をまとめて、あ、やっぱり投資家にメールを送ります",
  "context": "morning_mit",
  "language": "ja"
}
```

**コンテキスト一覧:**
```
morning_mit    → 朝の最優先事項設定（箇条書き抽出）
daytime_memo   → 昼のメモ（要約 + タスク抽出）
evening_done   → 夜の「できたこと」（成果を整理）
evening_thanks → 夜の「感謝」（感謝の対象を整理）
evening_plan   → 夜の「明日の予定」（箇条書き抽出）
free_text      → 自由入力（一般的な整形のみ）
```

**レスポンス:**
```json
{
  "cleaned": "今日は企画書をまとめて、投資家にメールを送ります。",
  "structured": {
    "tasks": ["企画書をまとめる", "投資家にメールを送る"],
    "summary": "企画書の完成とビジネスメール送信"
  },
  "changes": [
    { "type": "filler_removed", "original": "えーと" },
    { "type": "filler_removed", "original": "あの" },
    { "type": "self_correction", "original": "あ、やっぱり", "kept": "投資家にメール" },
    { "type": "punctuation_added", "position": 15 }
  ]
}
```

## Claude APIプロンプト設計

### 基本プロンプト（全コンテキスト共通）

```
あなたは音声入力テキストの整形AIです。
以下のルールに従って、話し言葉を読みやすい文章に整形してください。

【必須処理】
1. フィラー除去: 「えーと」「あのー」「うーん」「まあ」「なんか」を削除
2. 言い間違い修正: 途中で言い直した場合、最終的な意図のみ残す
3. 句読点追加: 適切な位置に「、」「。」を追加
4. 改行整形: 意味の区切りで改行
5. 誤字修正: 明らかな音声認識の誤変換を修正

【禁止事項】
- 意味を変えない
- 情報を追加しない
- 過度に丁寧にしない（話者のトーンを維持）
```

### コンテキスト別追加プロンプト

**morning_mit（朝の最優先事項）:**
```
追加指示: 入力からタスクを抽出し、箇条書きにしてください。
最大3つ。動詞で終わる形式に統一。
出力形式: { "cleaned": "...", "tasks": ["...", "..."] }
```

**evening_done（できたこと）:**
```
追加指示: 達成したことを整理し、成果として読みやすくまとめてください。
出力形式: { "cleaned": "...", "achievements": ["...", "..."] }
```

**evening_thanks（感謝）:**
```
追加指示: 感謝の対象と理由を明確にしてください。
出力形式: { "cleaned": "...", "gratitude": [{"to": "...", "reason": "..."}] }
```

**evening_plan（明日の予定）:**
```
追加指示: 明日のタスクを抽出し、箇条書きにしてください。
出力形式: { "cleaned": "...", "tasks": ["...", "..."] }
```

## データフロー（1日の流れ）

```
朝 7:00
  通知「おはよう！今日の最優先事項は？」
    ↓
  🎤「えーと今日は企画書まとめて、あと投資家にメール送って、
     あー市場調査もしなきゃ」
    ↓
  AI整形 → タスク抽出
    ↓
  最優先事項:
    1. 企画書をまとめる
    2. 投資家にメールを送る
    3. 市場調査をする
    ↓
  [OK] → DB保存

昼 12:00
  アプリを開く
  「順調か？」
    ↓
  🎤「企画書は半分できた。メールはまだ書けてない」
    ↓
  AI整形 → メモとして保存
    ↓
  進捗メモ: 企画書は半分完成。メールは未着手。

夜 21:00
  通知「今日どうだった？」
    ↓
  🎤「企画書完成した！投資家にもメール送れた。
     田中さんが資料手伝ってくれてありがたかった。
     明日は市場調査の続きと、プレゼン資料作りたい」
    ↓
  AI整形 → 自動分類
    ↓
  できたこと: 企画書完成、投資家メール送信
  感謝: 田中さん（資料作成の手伝い）
  明日の予定: 市場調査の続き、プレゼン資料作成
  スコア提案: 仕事 80（2/3達成）
    ↓
  [保存] → 日記としてDB保存
```


### voice-ai.ts

```typescript
type VoiceContext =
  | "morning_mit"
  | "daytime_memo"
  | "evening_done"
  | "evening_thanks"
  | "evening_plan"
  | "free_text";

type VoiceResult = {
  cleaned: string;
  tasks?: string[];
  achievements?: string[];
  gratitude?: { to: string; reason: string }[];
  changes: {
    type: "filler_removed" | "self_correction" | "punctuation_added" | "typo_fixed";
    original?: string;
    kept?: string;
  }[];
};

async function processVoice(
  rawText: string,
  context: VoiceContext
): Promise<VoiceResult>
```

### /api/voice/route.ts

```typescript
POST /api/voice
  → rawText + context を受け取り
  → Claude API (Haiku) を呼び出し
  → 整形結果を返却
```

## Typelessとの差別化ポイント

| 項目 | Typeless | 夢ナビ Voice AI |
|------|----------|----------------|
| 音声整形 | テキストを整えるだけ | **コンテキスト別に構造化** |
| タスク抽出 | なし | **自動でタスク分類** |
| 感情分析 | なし | **感謝・成果を自動分類** |
| 変更表示 | なし | **何を変えたか表示** |
| コスト | 月$12〜30 | **月約8円** |
| オフライン音声認識 | 不可 | **Web Speech API = ローカル** |
| 目的 | 汎用テキスト | **人生の目標達成に特化** |
| セッション制限 | 6分 | **制限なし** |
| 日記蓄積 | なし | **全記録が人生の財産** |
